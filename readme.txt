Структура проєкту

	•	app.py:
	        Головний файл для запуску бота.
	•	banners/:
	        Містить зображення та банери, що використовуються ботом.
	•	common/:
	        Загальні модулі та утиліти, спільні для всього проєкту.
	•	database/:
	        Файли бази даних та скрипти для роботи з нею.
	•	filters/:
	        Фільтри для обробки вхідних повідомлень та команд.
	•	handlers/:
            Обробники подій та команд від користувачів.
	•	kbds/:
	        Модулі, що відповідають за клавіатури та кнопки в інтерфейсі бота.
	•	middlewares/:
	        Проміжні шари для обробки запитів та відповідей.
	•	tests/:
	        Тестові сценарії для перевірки функціональності бота.
	•	utils/:
	        Додаткові утиліти та допоміжні функції.
	•	.circleci/:
	        Налаштування для безперервної інтеграції з використанням CircleCI.
	•	.gitignore:
	        Файл, що визначає файли та директорії, які ігноруються Git.
	•	identifier.sqlite:
	        Файл бази даних для зберігання ідентифікаторів.
	•	my_base.db:
	        Основна база даних проєкту.
	•	pytest.ini:
	        Конфігураційний файл для Pytest.
	•	requirements.txt:
	        Список залежностей та необхідних пакетів для проєкту.
